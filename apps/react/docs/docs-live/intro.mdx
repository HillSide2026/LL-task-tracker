# Levine LLP Matter Platform

The Levine LLP matter platform supports internal matter opening, maintenance, task tracking, document handling, and workflow-driven operational controls.

The authoritative deployment path is:

```sh
docker compose \
  --env-file deployments/levinellp/.env \
  -f deployments/levinellp/docker-compose.camunda7.yml \
  up -d --build
```

Use `deployments/levinellp/README.md` as the operational source of truth.

## Current Runtime

- Public portal: `https://matters.levinellp.ca`
- Object storage API: `https://s3.matters.levinellp.ca`
- Keycloak: `https://matters.levinellp.ca/auth`
- Case API: `https://matters.levinellp.ca/api`
- Storage API: `https://matters.levinellp.ca/storage`

The deployment stack uses Camunda 7, MongoDB, Keycloak, MinIO, OPA, Traefik, Java services, and the React case portal.

## Operating Boundary

This docs site covers the Levine LLP deployment and operations surface. Inherited upstream platform documentation remains in the repository for reference, but it is not part of the published Levine documentation set.
